Capabilities

Everything a coding agent needs

From file operations to OSINT investigations — Nexus Agent ships with a comprehensive tool ecosystem designed for real-world software engineering.

📁

File Operations

Read, write, and edit files with surgical precision. Supports diff-based editing, batch operations, and syntax-aware display.

🔍

Code Search & Discovery

grep, glob, and list_dir tools for rapid codebase exploration. Find patterns, navigate directories, and understand project structure.

⚡

Shell Execution

Run commands, tests, and builds directly. Built-in safety policies with dangerous command detection and approval workflows.

🌐

Web & Network

Web search, web fetch, URL scraping, and browser automation. Gather information from the web programmatically.

🛡️

OSINT Investigation

Domain, email, IP, phone, username, WHOIS, Censys, Shodan, and social media intelligence gathering tools.

📄

RAG — PDF Querying

Ingest PDF documents and query them using vector search. Ask natural language questions about your document collection.

🔗

MCP Integration

Model Context Protocol support — connect external MCP servers via stdio or SSE. Extend the agent with custom tools dynamically.

🧩

Sub-Agents

Spawn focused child agents for codebase investigation and code review. Each runs with isolated context and limited tool access.

🔐

Safety & Approval

6 approval policies (on-request, auto, never, yolo, etc.). Regex-based dangerous/safe command detection. Path traversal protection.

🪝

Hook System

Trigger external scripts at 5 lifecycle events: before/after agent, before/after tool, on error. Context passed via environment variables.

🧠

Context Management

Smart context compression at 80% token threshold. LLM-summarized conversation history, tool output pruning, loop detection.

🖥️

Dual UI — TUI + Web

Rich terminal TUI with semantic theming, plus a FastAPI + WebSocket web UI with a dark-themed SPA. Same agent, two interfaces.

Architecture

Event-driven agentic loop

The entire system is built around an async generator pipeline — user input flows through the agent, LLM, tools, and events stream back to the UI.

→

→

AgentEvent stream

↔

🔄

Agentic Loop

Streaming LLM responses → parallel tool execution → context compression → loop detection. Configurable max turns and retry logic.

📦

Context Window

256K token context with proactive compression at 80% threshold. LLM-summarized history with structured sections.

🔄

Loop Detection

Signature-based cycle detection (20-entry deque). Identifies exact repeats and A-B-A-B patterns, injects loop-breaker prompts.

📐

Pydantic Validation

Every tool uses Pydantic BaseModel for parameter schemas. Automatic JSON schema → OpenAI function-calling format.

Webhook System

Lifecycle event hooks

Nexus Agent fires webhook-style events at every stage of the agent lifecycle — before/after the agent runs, before/after every tool call, and on error. External scripts receive context via environment variables.

Hook Event Reference

Trigger Timing Env Variables Set Use Case
before_agent Before agent processes user message NEXUS_AGENT_TRIGGER, NEXUS_AGENT_USER_MESSAGE Logging, telemetry, rate limiting
after_agent After agent completes response NEXUS_AGENT_TRIGGER, NEXUS_AGENT_RESPONSE Notifications, audit logs, CI triggers
before_tool Before each tool execution NEXUS_AGENT_TOOL_NAME, NEXUS_AGENT_TOOL_PARAMS Approval gating, pre-flight checks
after_tool After each tool completes NEXUS_AGENT_TOOL_RESULT, NEXUS_AGENT_TOOL_NAME Result caching, metrics, alerting
on_error On agent or tool exception NEXUS_AGENT_ERROR Error reporting, fallback actions
Configuration

Flexible & hierarchical

Config loads from three levels — system-wide, project-level, and Agent.md — merged with a deep-merge strategy. Secrets come from environment variables.

~/.config/nexus-agent/config.toml
# Nexus Agent Configuration

[model]
name = "nvidia/nemotron-3-super-120b-a12b:free"
temperature = 1.0
context_window = 256_000

[shell_environment]
exclude_patterns = ["*KEY*", "*TOKEN*", "*API*", "*SECRET*"]

approval = "on-request"
max_turns = 100

[mcp_server.mytools]
command = "node"
args = ["server.js"]

[[hooks]]
name = "notify"
trigger = "after_agent"
command = "python3 notify.py"
timeout_sec = 30
